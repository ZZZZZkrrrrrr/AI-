# AI 视频获客运营平台阶段计划

## 项目定位

本项目不只是单点的视频生成工具，最终目标是做成一套「AI 内容生产 + 多平台分发 + 线索获客 + 私域转化」的聚合平台。

当前已经完成的 AI 视频生成工作流，是整个系统的第一层能力。后续阶段需要在现有工作流上逐步叠加批量生产、素材管理、平台分发、评论线索、企业微信承接和运营数据分析。

整体路线建议采用分阶段交付，不一次性做大而全，优先跑通业务闭环。

```text
内容生产
→ 批量生成
→ 视频拼接
→ 多平台分发
→ 评论线索采集
→ AI 客服/企业微信承接
→ 私域转化与复盘
```

## 第一阶段：AI 内容生产工作台

### 阶段目标

先把「提示词包 + 商品图 → AI 分析 → 最终提示词 → libTV 视频生成 → 任务状态回写」这条主链路做稳定。

这个阶段解决的是视频从 0 到 1 的生产问题，重点是可用、可追踪、可复现。

### 核心功能

- 单条视频生成
- Word 提示词包上传
- 中文提示词包解析
- 商品图片上传
- 图片识别品类、主体、卖点、风险点
- 自动生成最终完整视频提示词
- 最终提示词可人工修改、复制、手动粘贴
- 写入 SQLite 数据库
- 提交 libTV 生成视频
- libTV 任务状态展示
- 生成结果回写数据库
- 视频命名包含序号、日期、类别
- 每一步显示状态、输出内容、完成时间
- 模型过程可查看
- 模型可在系统设置中切换

### 当前状态

当前项目已经基本完成这一阶段的核心链路，包括：

- 提示词工作台
- 单条视频生成
- 批量生成入口
- 视频拼接入口
- libTV 桥接
- SQLite 数据库记录
- 任务看板
- 素材与输出查看
- 系统设置
- 深色/浅色主题切换
- token 用量展示
- 中断生成能力

### 验收标准

- 上传一个提示词包和一张商品图，可以完整生成最终提示词。
- 可以选择先验证或真实提交 libTV。
- 生成过程每一步都有状态、时间和输出。
- 数据库中可以查到产品、提示词、任务、视频结果。
- 生成失败时可以看到失败原因。

## 第二阶段：批量生产中心

### 阶段目标

把单条视频生产升级为批量任务生产，支持一天生成 300 条到 1000 条视频的生产能力。

这个阶段解决的是规模化生产问题。

### 核心功能

- 多个 Word 提示词包批量上传
- 多张商品图批量上传
- CSV / Excel 导入任务配置
- 文件名一对一匹配
- 一个提示词包匹配多张图
- 多提示词包和多商品图交叉生成
- 批量任务预检
- 错误行提示
- 只提交有效任务
- 批次列表
- 批次状态统计
- 并发数配置
- 任务排队
- 自动重试
- 失败任务单独重跑
- 批量生成后自动提交 libTV
- 批量成本、耗时、token 统计

### 推荐设计

批量生成不要直接一次性把所有任务推给模型和 libTV，而是做成队列：

```text
上传任务包
→ 生成待处理任务表
→ 按并发数领取任务
→ 单条任务执行完整链路
→ 写入状态
→ 失败任务进入重试或人工处理
```

### 验收标准

- 上传 10 条测试任务，可以自动生成 10 条任务列表。
- 可以勾选部分任务启动。
- 执行过程中可以看到每条任务的状态。
- 失败任务不会影响整个批次继续执行。
- 可以按照批次查看最终视频结果。

## 第三阶段：视频拼接与素材中心

### 阶段目标

把生成出来的视频变成可组合、可复用的素材资产。

这个阶段解决的是视频后处理和素材管理问题。

### 核心功能

- 展示已生成视频
- 勾选多个视频进行拼接
- 支持按顺序拼接
- 支持调整排序
- 支持选择画幅
- 支持输出拼接视频
- 使用 ffmpeg 合成
- 拼接任务入库
- 拼接结果保存到输出目录
- 素材按类别、日期、任务编号筛选
- 支持下载视频
- 支持查看原始任务和最终提示词

### 验收标准

- 可以从已生成视频中勾选 2 个以上视频。
- 点击合成后生成一个完整拼接视频。
- 拼接任务有状态、开始时间、结束时间和输出路径。
- 拼接失败时显示 ffmpeg 错误原因。

## 第四阶段：多平台分发

### 阶段目标

把生成好的视频发布到抖音、快手、小红书、视频号等平台。

这个阶段解决的是内容分发问题。

### 实现路线

优先级建议：

1. 官方 API
2. 半自动发布
3. RPA / 手机自动化

如果平台官方 API 支持发布，优先走 API；如果不开放，再考虑 RPA 或手机自动化。

### 核心功能

- 平台账号管理
- 发布素材选择
- 标题、文案、话题配置
- 定时发布
- 发布状态回写
- 发布失败原因记录
- 多平台发布记录
- 平台链接回写
- 按账号查看发布历史

### 风险点

- 平台风控
- 账号安全
- 验证码
- 登录态失效
- 高频发布限制
- RPA 脚本受页面变化影响

### 验收标准

- 至少支持一个平台完成发布闭环。
- 可以查看发布成功或失败状态。
- 发布记录可以追溯到原始视频任务。

## 第五阶段：评论线索采集与意向识别

### 阶段目标

把视频发布后的评论、私信、互动数据变成可跟进的销售线索。

这个阶段解决的是获客问题。

### 核心功能

- 采集评论
- 采集私信或互动数据
- 按视频、平台、账号归档
- AI 判断用户意向
- 自动打标签
- 高意向线索提醒
- 生成回复建议
- 支持人工确认
- 线索进入跟进池

### 线索分层

- 强意向：明确询价、咨询购买、要求联系方式
- 中意向：表达兴趣、询问细节、收藏点赞活跃
- 弱意向：泛泛评论、情绪表达
- 无效线索：广告、无关内容、重复用户

### 风控建议

自动点赞、自动私信、自动加微信要谨慎。建议第一版采用「AI 建议 + 人工确认」模式，先降低平台风控风险。

### 验收标准

- 可以按视频查看评论线索。
- AI 能给出意向等级。
- 运营人员可以确认、忽略或加入跟进。

## 第六阶段：企业微信与私域转化

### 阶段目标

把公域线索承接到企业微信、社群和客服系统中。

这个阶段解决的是转化和客户关系管理问题。

### 核心功能

- 企业微信账号绑定
- 线索同步
- 客户标签
- 跟进状态
- AI 回复建议
- 标准话术库
- 群运营记录
- 客户阶段管理
- 转化结果记录
- 二次触达提醒

### 私域流程

```text
评论/私信线索
→ AI 判断意向
→ 人工确认
→ 引导添加企业微信
→ 打标签
→ 进入社群或一对一跟进
→ 记录转化结果
```

### 验收标准

- 线索可以进入企业微信跟进表。
- 每个客户有来源、标签、跟进状态。
- 能查看从哪个视频、哪个平台带来的客户。

## 第七阶段：数据看板与成本核算

### 阶段目标

让业务方知道系统到底生产了多少内容、花了多少钱、带来了多少线索和转化。

### 核心功能

- 视频生成数量
- 成功率
- 失败率
- 平均耗时
- token 用量
- 模型调用成本
- libTV 生成成本
- 平台发布数量
- 评论数
- 线索数
- 强意向线索数
- 转化数
- ROI 粗算

### 验收标准

- 能按天、周、月查看生产和转化数据。
- 能看出哪个品类、账号、平台效果更好。
- 能判断模型成本是否可控。

## 第八阶段：SaaS 化与聚合平台

### 阶段目标

把内部工具升级为多用户、多租户、可商业化的平台。

### 核心功能

- 用户登录
- 权限管理
- 团队空间
- 项目管理
- 模型通道管理
- API Key 管理
- 余额和计费
- 任务额度
- 审计日志
- 操作日志
- 服务器部署
- 域名访问
- 数据备份
- 异常告警

### 平台形态

最终平台可以拆成几个中心：

- 内容生产中心
- 批量任务中心
- 素材中心
- 分发中心
- 线索中心
- 私域运营中心
- 模型与成本中心
- 系统设置中心

### 验收标准

- 多个用户可以同时使用。
- 每个用户只能看到自己的项目和数据。
- 任务、成本、素材、客户数据可以隔离。
- 平台可以部署到服务器和域名上。

## 开发优先级建议

### 近期优先

1. 稳定单条视频生成链路。
2. 完善批量任务队列。
3. 做好视频拼接。
4. 完成素材与输出管理。
5. 增加成本和 token 统计。

### 中期优先

1. 多平台发布。
2. 评论采集。
3. 线索意向识别。
4. 企业微信承接。

### 后期优先

1. 多租户。
2. 计费系统。
3. API 聚合网关。
4. RPA 手机矩阵。
5. SaaS 产品化。

## 关键原则

- 先跑通闭环，再追求全自动。
- 先半自动，再逐步自动化。
- 能用 API 的地方优先 API。
- RPA 作为补充方案，不作为唯一基础。
- 每个任务必须有状态、日志、成本和结果。
- 平台要围绕「内容生产效率」和「线索转化效果」两个核心指标建设。
