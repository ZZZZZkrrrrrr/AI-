# 手机端 App 发布前自检清单

日期：2026-06-13
适用项目：`01_apps/ai_prompt_video_studio`
当前目标：让不会技术的运营、编导和商家用户，打开手机端后能在 30 秒内知道先点哪里、上传什么、等待多久、结果去哪里拿。

## 一句话目标

手机端不是缩小版后台，而是“创作 App”。首屏要先给入口，生成中要给等待反馈，结果页要能直接下载，设置页不能暴露技术路径和代码。

## 发布前必须通过

| 模块 | 必须达到的效果 | 不允许出现 |
| --- | --- | --- |
| 首页 | 大按钮入口清楚，首屏能看到常用工作栏和热门模板 | 首屏堆太长、像后台数据页 |
| 创意页 | 用户能从模板开始，看到上传图片要求，提交后知道正在处理 | 用户重复点击生成、看不到等待状态 |
| 任务页 | 视频生成时显示动态等待、预计剩余时间和进度 | 只有静态“处理中” |
| 资产页 | 素材、图片、视频都有下载入口，空状态告诉用户下一步 | 显示本机数据库路径、素材路径、内部文件名 |
| 我的页 | 提供开始创作、素材结果、任务进度、反馈问题的快捷入口 | 空白列表、技术配置、原始 JSON |
| 右滑侧栏 | 手机任意页面右滑都打开侧栏，侧栏跟随手指移动 | 浏览器返回预览、侧栏断层、重复品牌头 |
| 黑白模式 | 手机端也能切换，白天模式颜色清爽、按钮有层级 | 白底上出现过重黑卡、文字灰到看不清 |
| PWA/App | 可安装、图标和启动页完整，基础离线提示正常 | 只像网页，不像可交付 App |

## 手机端首屏标准

1. 顶部只保留必要操作：菜单、状态、通知、主题、退出。
2. 首页第一屏优先展示“开始创作”和 3 到 5 个常用入口。
3. 非核心后台信息折叠到二级页面，不占首屏。
4. 热门玩法和模板可以横向滑动，不能把页面拉得很长。
5. 所有主按钮文案用用户语言，例如“开始创作”“保存视频”“查看进度”，少用“数据库”“任务 ID”“素材路径”。

## 创意页标准

1. 模板卡必须写清楚：这是“一键生成女装轻奢室内空间 15 秒杂志感穿搭氛围视频”的模板，不是通用模板。
2. 上传图片要求必须在模板里直接看到：
   - 建议上传成年女性人物参考图、女装服装或商品图。
   - 有封面图、详情页头图、品牌画报图时可以一起上传做风格参考。
   - 图片要能清楚看到颜色、版型、领口、腰线、裙摆、面料纹理。
   - 避免模糊、遮挡、强滤镜、拼图和带水印图片。
3. 封面图在模板卡上要真实显示，不使用空渐变占位。
4. 点击生成后按钮进入等待状态，显示“等待结果”或“排队中”。
5. 已提交后要提示用户可以去任务页查看，不要让用户误以为没点成功。

## 视频生成等待标准

1. 手机端和网页端都要显示动态等待反馈。
2. 等待信息至少包含：
   - 当前阶段，例如“已提交”“排队中”“生成中”“准备下载”。
   - 预计剩余时间。
   - 进度百分比或进度条。
3. 不同秒数视频可以按经验估算，但必须让用户知道系统还在工作。
4. 等待期间不能鼓励重复提交。
5. 失败时要给下一步，例如“重试”“重新提交”“去素材页查看已有结果”。

## 资产页标准

1. 图片来源、素材记录、视频输出都要有保存或下载按钮。
2. 下载按钮点击后要有反馈，例如“正在保存”“已开始下载”。
3. 空状态不能只写“暂无内容”，要告诉用户下一步：
   - 去创意页选模板。
   - 上传商品图或封面图。
   - 生成后回到这里下载。
4. 手机端不显示本机路径、数据库路径、内部 JSON、用户 ID、原始任务 ID。
5. 文件名可以简化成“提示词文档”“商品图片”“成片视频”这类用户能理解的名字。

## 我的页标准

1. 顶部只显示当前账号和常用入口，不重复展示品牌头。
2. 设置内容分成小白能理解的卡片：
   - App 准备中心
   - 问题反馈
   - 账号数据
   - 协议与说明
3. 空列表必须有操作引导，不能只留白。
4. 管理员信息只能管理员看到，普通用户不暴露审核队列。
5. 技术诊断信息默认折叠，需要时再打开。

## 右滑侧栏标准

1. 从屏幕左侧向右滑时，侧栏必须跟随手指移动。
2. 已打开侧栏时，向左滑可以自然关闭。
3. 横向模板滑动区不应误触发侧栏，除非从屏幕边缘开始滑。
4. 白天模式和黑夜模式下侧栏宽度、遮罩、阴影要一致。
5. 创意页、首页、资产页、我的页都要表现一致。

## 发布检查命令

在 `01_apps/ai_prompt_video_studio` 下执行：

```bash
npm run mobile:release-checklist
npm run mobile:check
npm run mobile:density
npm run text-image:check
npm run pwa:check
npm run build
```

## 当前验收口径

| 检查 | 通过标准 |
| --- | --- |
| `mobile:release-checklist` | 关键手机端体验点全部存在 |
| `mobile:check` | 新手友好、侧栏、下载、等待、空状态等检查通过 |
| `mobile:density` | 手机端按钮和卡片高度不过大 |
| `text-image:check` | 文生图入口、尺寸、提示词、结果处理正常 |
| `pwa:check` | App 安装相关资源完整 |
| `build` | 能正常打包，主包不超过当前预算 |

## 下一轮优先级

1. 继续检查创意页右滑侧栏在白天模式下是否还有遮罩断层。
2. 继续压缩首页和创意页首屏高度，让常用入口更靠前。
3. 给更多模板补封面图、上传要求和“一键用途”说明。
4. 给生成失败增加更明确的“下一步按钮”。
5. 做一轮真机截图证据，把 390px、430px、微信内置浏览器、PWA 安装后入口都截图留档。
