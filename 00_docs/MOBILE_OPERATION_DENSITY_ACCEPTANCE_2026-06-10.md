# 手机端操作界面尺寸验收标准

日期：2026-06-10  
适用项目：`01_apps/ai_prompt_video_studio`  
当前状态：已接入自动化检查 `npm run mobile:density`

## 背景

根据 2026-06-10 用户提供的手机录屏，当前手机端已经具备“小白友好大按钮”的方向，但操作界面存在卡片偏高、圆形图标偏大、底部导航占用过多竖向空间的问题。优化目标不是把按钮变小到难点，而是让 390px 和 430px 宽度的手机首屏能露出更多真实操作入口。

## 本轮尺寸基线

| 区域 | 当前基线 | 验收原因 |
| --- | --- | --- |
| 首页操作卡片 | `128px` 最小高度 | 保留大按钮感，同时减少无效空白 |
| 首页圆形图标 | `54px` 宽高 | 图标仍清楚，不抢占文字和第二排入口空间 |
| 生产总览状态卡 | `92px` 最小高度 | 小屏保持双列，减少总览滚动负担 |
| 创建页步骤切换 | `40px` 最小高度 | 保持可点，同时降低顶部操作条高度 |
| 创建页输入步骤卡 | `46px` 最小高度 | 一屏能看到更多输入步骤 |
| 模型通道卡 | `112px` 最小高度 | 保留状态信息，但不做厚重卡片 |
| 底部导航项 | `46px` 最小高度 | 仍满足手指点击，不再压住内容 |
| 页面底部留白 | `78px + safe-area` | 给底部导航留空间，同时避免多余空白 |

## 设计规则

1. 首页继续使用两列操作卡片，不改成小图标宫格。这个产品面向非技术用户，入口必须像“任务按钮”，不是后台菜单。
2. 卡片高度应保持在 `112px` 到 `136px` 之间。低于这个范围会显得拥挤，高于这个范围会回到录屏里的低密度问题。
3. 圆形图标应保持在 `52px` 到 `58px` 之间。图标颜色用于区分任务类型，但不替代文字说明。
4. 生产总览和批量统计在小屏使用双列状态卡，不再强制单列堆叠。
5. 创建页顶部的步骤切换和输入步骤卡要服务“继续操作”，不能像桌面表单一样占据大半屏。
6. 底部导航必须保留 `safe-area-inset-bottom`，但导航本身不能超过必要高度。
7. 所有紧凑化都不能牺牲可点区域：核心按钮不低于 40px，底部导航不低于 44px。

## 自动化门禁

新增命令：

```bash
npm run mobile:density
```

该命令检查：

- 首页操作卡片高度是否在紧凑范围内。
- 首页圆形图标是否保持清楚但不过大。
- 小屏生产总览状态卡是否保持双列和较低高度。
- 创建页步骤切换和输入步骤卡是否保持低高度。
- 模型通道卡是否不过厚。
- 底部导航项和页面底部安全区是否合理。
- QA 计划是否包含 `operation-density` 用例组。
- 截图证据和本文档是否存在。

`npm run app:health` 已接入 `mobile:density`，因此发布总检查会自动覆盖这项尺寸门禁。

## 不通过时怎么改

如果用户再次反馈“操作界面太大”，优先检查：

1. `src/styles.css` 里的 `.mobile-action-tile` 是否超过 `136px`。
2. `.mobile-bottom-nav-item` 是否超过 `48px`。
3. 小屏 `.overview-status-card` 是否被改回单列或超过 `100px`。
4. 创建页 `.mobile-input-tabs button` 是否超过 `50px`。
5. 是否新增了大面积说明文字、提示卡或营销式模块挤占首屏。

如果用户反馈“按钮太小”，优先检查：

1. 首页操作卡片是否低于 `112px`。
2. 底部导航是否低于 `44px`。
3. 创建页核心按钮是否低于 `40px`。
4. 标签、徽标和长标题是否换行后遮挡图标或副标题。
