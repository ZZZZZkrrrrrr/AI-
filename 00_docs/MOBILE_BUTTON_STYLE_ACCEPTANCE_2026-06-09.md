# 手机端小白按钮风格验收标准
日期：2026-06-09  
适用项目：`01_apps/ai_prompt_video_studio`

## 目标

参考用户给出的“大按钮 + 圆形图标 + 分段切换 + 底部导航”手机界面后，本项目的手机端首页应优先像一个可直接开始创作的 App，而不是一个缩小版后台。

验收重点不是照搬娱乐化视觉，而是保留对小白友好的信息结构：

- 首屏先回答“我现在点哪里开始”，再展示系统状态。
- 每个大按钮只表达一个动作，不使用后台模块名做主标题。
- 图标帮助识别，文字帮助决策。
- PWA 安装提示、模型状态、libTV 状态和系统看板不能压过核心创建入口。
- 所有按钮必须进入真实页面或真实流程，不能做成未接入的假入口。

## 当前已落地的手机端按钮结构

首页移动端已使用 `mobile-beginner-launcher`：

- 顶部文案：`新手模式 / 你想先做什么？`
- 分段切换：`图文 / 视频`
- `图文` 模式按钮：
  - `上传商品图`
  - `整理素材`
  - `提示词包`
  - `我的数据`
- `视频` 模式按钮：
  - `一键做视频`
  - `批量生成`
  - `图生视频`
  - `视频拼接`
  - `素材结果`
  - `我的数据`
- 底部导航：`首页 / 创建 / 素材 / 我的`

这些入口分别连接到现有 `studio`、`batch`、`stitch`、`assets`、`settings` 页面，避免了“按钮看起来能点，但没有真实功能”的上架风险。

其中创建类按钮不应只把用户丢到同一个复杂创建页。当前已支持 `studio:` 创建意图：

- `studio:images`：进入创建页后默认打开 `商品图` 步骤。
- `studio:prompt`：进入创建页后默认打开 `提示词包` 步骤。

因此 `上传商品图`、`一键做视频`、`图生视频` 会先落到商品图步骤，`提示词包` 会直接落到提示词包步骤。

## 视觉规则

- 大按钮使用双列网格，一屏优先露出 4 个核心动作。
- 每个按钮最小高度保持在 112px 到 136px，保证拇指点击和视觉识别，同时让首屏露出更多入口。
- 圆形图标建议 52px 到 58px，使用不同色相区分动作，不做单一色系堆叠。
- 主标题控制在 2-5 个汉字或短词，副文案只说明结果，例如“图片变成短片”。
- `推荐`、`HOT`、`MAX` 这类标签只用于引导优先级，不能替代真实功能说明。
- 按钮圆角控制在 8px 左右，保持工具型产品的稳定感。

## 交互规则

- 首次打开手机端时，PWA 安装提示排在新手入口之后，不打断用户理解核心功能。
- 分段切换必须是 44px 以上可点区域，并在首页内切换按钮组，不能点一下就离开首页。
- 创建类按钮必须能带入具体步骤意图，例如商品图入口打开商品图步骤，提示词包入口打开提示词包步骤。
- 底部导航固定在底部，并留出 `safe-area-inset-bottom`。
- 桌面宽表格在手机端必须切换为任务卡片或素材卡片。
- 设置页必须保留隐私政策、帮助支持、数据导出和删除账号入口，不能因为“简化”而隐藏合规入口。
- 2026-06-12 调整后，“我的”页手机端允许把试用反馈、数据与账号、合规与协议、模型和运行状态默认折叠；但入口必须仍在同一页可展开，不能移除。

## 机器验收

`npm run mobile:check` 现在会检查：

- 新手入口是否存在。
- 小白文案是否存在。
- 图文模式和视频模式的大按钮是否都存在。
- 图文/视频分段是否在首页内切换，而不是直接跳页。
- `studio:` 创建意图是否存在，并能映射到创建页的手机端步骤。
- 大按钮是否通向真实页面。
- 双列按钮网格、圆形图标、大触控高度是否存在。
- 图文/视频分段是否满足触控高度。
- PWA 安装提示是否排在新手入口之后。
- 底部导航是否留出安全区。
- 任务、素材、视频拼接页是否隐藏桌面表格并使用手机卡片。

2026-06-12 复核结果：

- `.mobile-action-grid` 已恢复为双列网格。
- `.mobile-action-tile` 最小高度为 112px。
- `.mobile-action-icon` 为 54px 圆形图标。
- `.mobile-mode-switch button` 最小高度为 44px。
- `npm.cmd run mobile:check` 通过：103 项通过，0 项失败，0 项警告。

`npm run screenshots:check` 继续作为视觉证据，确认 390px、430px 手机截图和 768px 平板截图存在并符合计划尺寸。

截图计划现在要求 `mobile-web-390` 同时覆盖默认视频模式首页、点击 `图文` 后的图文模式首页，以及从图文模式点击 `提示词包` 后进入创建页提示词包步骤的链路。捕获脚本支持结构化 `clickText` 交互，截图前会自动点击指定文本，避免只保存默认状态而漏掉分段切换证据。

## 官方资料映射

- Apple Human Interface Guidelines: Buttons 和 Tab bars 强调按钮应表达清晰动作，底部标签栏承载主要导航。
- Google Play 账号删除要求要求有应用内删除路径和网页删除入口，因此“我的数据”不能隐藏。
- PWA 可安装性资料要求 manifest、图标和安装入口完整，但安装引导不应遮挡核心使用路径。

## 后续迭代

下一步建议优先做三件事：

1. 给 `mobile-web-390` 首页截图增加真实演示数据，让按钮下方能露出最近结果或任务进度。
2. 在创建页继续压缩表单，只保留商品图、卖点、风格三个小白必填项，高级设置折叠。
3. 准备 App Store / Google Play 审核截图文案时，把第一张截图定位为“打开就能开始做视频”，而不是“后台总览”。
