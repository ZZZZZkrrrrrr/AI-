# AI 生成内容标识与证据包方案

日期：2026-06-09  
适用项目：`01_apps/ai_prompt_video_studio`
最近更新：2026-06-11，补充文生图账号级画布节点的证据边界。

## 1. 本轮已落地

- 单条视频创建页新增 `AI 证据包` 下载按钮。
- 证据包文件名格式：`{商品名或任务ID}-AI生成内容证据包-{日期}.json`。
- 证据包版本：`ai-disclosure-evidence-v1`。
- 已将证据包构建和 JSON 下载逻辑抽到 `src/shared/compliance/aiEvidencePack.js`，避免继续堆进 `App.jsx`。
- 证据包模块现在通过动态 import 按需加载，只在用户点击下载证据包时进入前端包，避免增加手机端首屏体积。
- `npm run compliance:check` 已增加检查，确保创建页保留 AI 证据包入口和生成函数。
- 证据包不包含原图 base64，不包含模型密钥，不包含服务端环境变量。
- 文生图模块已新增账号级画布节点，节点记录生成图片、提示词、模型、尺寸和生成时间；当节点送入单条视频时，证据包会记录节点来源。

## 2. 证据包包含内容

### 法务和标识信息

- 隐私政策 URL。
- 用户协议 URL。
- AI 生成内容说明 URL。
- 推荐显式标识：`AI 生成/AI 辅助生成`。
- 用户必须人工复核。
- 用户不得删除、篡改、伪造或隐匿 AI 标识。

### 任务信息

- `runId`
- 商品名称。
- 商品类目。
- 目标时长。
- 视频画幅。
- libTV 模式。
- 视频结果 URL。

### 输入摘要

- 提示词包文件名。
- 提示词正文长度。
- 商品补充信息摘要。
- 图片数量。
- 图片文件名、类型和大小。

不会导出图片 base64，避免证据包过大，也降低隐私泄露风险。

### 输出摘要

- 最终提示词长度。
- 最终提示词预览。
- 图片分析结果摘要。
- 识别类目。
- 提示词步骤数量。
- 视频步骤数量。

### 模型通道

- 分析模型。
- 视觉模型。
- 视频模型。
- 文生图模型。

### 文生图画布来源

当单条视频或批量任务使用了文生图画布里的图片时，证据包需要额外记录：

- `textImageCanvasNodeId`
- 图片文件名和输出 URL。
- 原始文生图提示词和反向提示词摘要。
- 文生图模型与尺寸。
- 图片节点生成时间。
- 图片进入视频流程的操作时间。

这能证明视频里使用的 AI 图片素材来自哪个账号、哪次生成、哪一个画布节点。

### 过程追溯

- 提示词生成步骤。
- 视频生成步骤。
- 最近 30 条模型事件摘要。
- Token 用量汇总。

### 人工复核清单

- 商品事实、价格、资质、授权和功效表达是否有证据支撑。
- 发布前是否保留平台要求的 AI 生成内容标识。
- 是否存在删除、篡改、伪造或隐匿 AI 标识的风险。
- 高风险领域是否人工复核。

## 3. 当前边界

当前证据包是“用户侧 JSON 证据包”，不是完整监管级归档：

- 证据包本体还没有写入服务端数据库或对象存储。
- 证据包本体还没有和具体 `video_tasks` 记录强绑定；但文生图图片来源已通过 `video_task_source_links` 绑定到 `task_code`。
- 文生图来源记录已可通过 `GET /api/task-source-links` 按当前账号查询，也会进入账号数据导出的 `textImageSourceLinks`，并已在“素材与输出”页展示。
- 没有写入视频文件元数据。
- 没有生成图片/视频水印。
- 没有生成平台发布字段。
- 没有管理员审核状态。
- 文生图画布节点已能随单条视频证据包记录 `textImageCanvasNodeId`、提示词摘要、模型、尺寸和送入时间；提交视频任务时，服务端也会把这类来源写入 `video_task_source_links`、`prompt_package.json` 和账号数据导出，前端素材页可直接查看。

这一步的价值是先让每次生成都有可下载、可复核、可交给运营的 AI 生成内容记录。

## 4. 正式发布前还需要补齐

1. 服务端保存证据包，并绑定 `task_id`、`owner_user_id`、`video_url`。
2. 视频结果页显示 AI 标识状态。
3. 下载视频时同时下载证据包。
4. 在对象存储写入证据包 JSON。
5. 对视频文件增加隐式标识或元数据字段。
6. 对导出文案和平台发布字段增加 AI 标识。
7. 管理员可查看、审核、锁定或重新生成证据包。
8. 把完整证据包归档到服务端或对象存储，并和 `task_id`、`task_code`、`owner_user_id`、`video_task_source_links` 关联。
