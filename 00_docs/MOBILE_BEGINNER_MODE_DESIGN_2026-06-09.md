# 手机端“小白模式”入口设计

日期：2026-06-09  
适用项目：`01_apps/ai_prompt_video_studio`

## 设计目标

参考用户提供的“AI 秒给”式入口后，本项目手机端不应直接照搬娱乐化界面，而应提炼出对新手最有帮助的部分：

- 用“大按钮”代替复杂后台菜单。
- 用“我要做什么”代替“系统模块名”。
- 用颜色和图标帮助识别任务类型。
- 把高级状态、数据库、libTV、模型配置放在二级页面或状态卡里。
- 保留底部导航，让用户随时回到首页、创建、素材、我的。

## 本轮已落地

手机端首页新增 `mobile-beginner-launcher`，桌面端不显示。入口包括：

- 一键做视频：进入单条视频创建。
- 批量生成：进入批量任务。
- 图生视频：进入图生视频/单条创建流程。
- 视频拼接：进入视频拼接。
- 素材结果：查看素材、任务和输出。
- 我的数据：进入帮助、隐私和数据权利入口。

样式特点：

- 浅色友好区域，降低后台工具的压迫感。
- 双列大按钮，适合拇指点击。
- 渐变圆形图标，用于快速区分功能。
- 推荐、HOT、MAX 等标签只用于引导，不替代真实功能说明。
- 所有入口仍连接现有业务路由，不新造假功能。

## 按钮式首页规范

如果继续向参考图的“小白友好按钮风格”靠近，建议采用下面的设计规范：

- 首屏先给选择，不先讲系统：手机首页顶部直接出现“你想先做什么？”和功能按钮，专业看板向下放。
- 每个按钮只表达一个动作：例如“一键做视频”“批量生成”“视频拼接”，避免“模型中心”“任务编排”这类后台词。
- 图标承担识别，文字承担决策：图标用圆形渐变底承载，文字保持黑色高对比，按钮内不要塞长说明。
- 两列网格是默认形态：一屏能看到 4 个左右主要动作，剩余动作继续向下滚动，不做横向滑动。
- 标签只标优先级：`推荐`、`HOT`、`MAX` 用于提示入口热度或套餐等级，不放在每个按钮上。
- 底部导航保持固定：`首页 / 创作 / 素材 / 我的` 是小白可理解的信息架构，专业模块放到二级页。
- 安装 App 提示不抢首屏：PWA 安装、更新、权限说明放在用户看到核心按钮之后。

本轮代码已把手机端快捷入口移动到首页内容最前，并让安装提示在移动端排到页面内容之后，避免第一次打开时被系统提示打断。

## 验证结果

已用生产构建重新抓取 `mobile-web-390` 与 `mobile-web-430` 首页截图。390px 手机首屏现在表现为：

- 顶部为压缩后的应用工具条，不再显示长说明文案。
- 首屏先出现“新手模式 / 你想先做什么？”。
- `图文 / 视频` 使用分段切换，视频态为渐变高亮。
- 主要功能为两列大按钮：圆形渐变图标、短标题、短说明、少量徽标。
- PWA 安装提示不再遮挡首页首屏。

同时修正了截图脚本的移动端仿真参数，避免截图时 CSS 像素被重复缩放，导致只截到页面左半边。

## 为什么这样设计

新手用户第一次打开手机端时，通常不会理解“提示词包、libTV、模型通道、SQLite、ffmpeg”等工程术语。首页应该先回答：

1. 我能做视频吗？
2. 我要点哪里开始？
3. 做完后去哪里看结果？
4. 出问题时去哪里找帮助？

因此手机端首页优先展示任务入口，而不是系统状态。系统状态仍保留，但向下放，避免抢占第一屏。

## 后续迭代建议

1. 增加“新手/专业”模式切换：新手看大按钮，专业看生产数据。
2. 增加演示数据：让截图和审核账号看到完整的商品图、生成进度和视频结果。
3. 增加空状态引导：没有任务时不要显示空表格，要显示“上传商品图 -> 生成提示词 -> 出视频”。
4. 增加一步式创建页：只保留商品图、商品卖点、视频风格三个输入，其余高级参数折叠。
5. 增加错误解释：失败时显示“额度不足 / 图片不清楚 / 模型未配置 / 视频生成排队中”，而不是技术错误。

## 设计边界

- 不建议把整个产品做成纯娱乐风格，因为它本质是生产工具。
- 不建议隐藏隐私、AI 生成说明和删除账号入口，应用商店审核需要看得见。
- 不建议使用过多动效，移动端首先要稳定、清晰、可重复操作。
- 不建议把未接入的能力包装成可用功能，所有按钮必须进入真实页面或明确说明状态。
