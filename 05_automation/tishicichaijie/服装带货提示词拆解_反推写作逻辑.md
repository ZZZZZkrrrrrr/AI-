# 服装带货提示词拆解：反推写作逻辑

拆解对象：

- `C:/Users/96259/Desktop/男性服装带货_15秒电影主角登场感视频提示词.docx`
- `C:/Users/96259/Desktop/女装服装带货_15秒场景氛围感视频提示词规范.docx`

## 一句话结论

这两个文件不是普通的“一条视频提示词”，而是一个完整的“提示词规范包”。

它们的写法可以概括成：

> 先限定视频风格和禁区，再让 AI 按步骤完成读图、造型、风格判断、场景匹配、摄影设计、15 秒分镜，最后收束成一条可直接复制的视频生成提示词。

男装包是原型：核心是“男性服装 + 电影主角登场 + 车辆身份氛围”。

女装包是在男装包骨架上迁移出来的版本：保留 10 步流程、9:16 竖屏、无口播、完整造型、摄影库、15 秒时间轴，但把核心卖点从“男人穿上像电影主角”改成“女生穿进真实生活场景后变漂亮、有氛围、可代入”。

如果要复用到所有商品，不要照搬“穿搭、造型、车辆、服装版型”这些服装专属词，而要复用它背后的通用结构：商品建档、用户购买心理、使用场景、卖点可视化、镜头时间轴、商品锁定规则和负面提示词。全品类版本见同目录的 `全品类商品提示词拆解_通用复用模型.md`。

## 这两个提示词包的底层公式

```text
提示词规范包 =
明确边界
+ 固定规则
+ 顺序流程
+ 专家角色
+ 输入信息表
+ 搭配/风格/场景/摄影词库
+ 15 秒分镜模板
+ 最终提示词模板
+ 负面提示词
+ 样片示例
```

它们不是靠灵感写出来的，而是靠“模块化约束”写出来的。每个模块都解决一个生成失控问题。

| 模块 | 解决的问题 | 写法特征 |
|---|---|---|
| 版本边界 | 防止 AI 跑到口播、直播间、剧情短剧、促销广告 | 开头直接声明“只服务一种风格” |
| 全局原则 | 防止输出形式混乱 | 固定时长、比例、声音、文字、人物、服装、场景、摄影 |
| 触发词流程 | 防止一步到位乱生成 | 设计【执行第一步】到【执行第十步】 |
| 图片读图建档 | 防止服装被改色、改款、改材质 | 先读取品类、颜色、版型、材质、结构、风险 |
| 造型师角色 | 防止只拍单品，人物造型不完整 | 让 AI 先当造型师，再当摄影指导 |
| 风格标签库 | 防止场景、道具、光影乱配 | 把服装归入稳定风格标签 |
| 气质主题 | 防止只写“帅”“漂亮”这种空词 | 写成一个可拍摄的人物状态 |
| 场景/车辆/道具库 | 防止画面没有生活用途或身份氛围 | 男装用车辆，女装用生活场景和携带物 |
| 摄影库 | 防止视频没有镜头语言 | 构图、景别、光影、运镜、摄影等级 |
| 时间轴 | 防止 15 秒没有节奏 | 固定 0-2、2-4、4-6、6-9、9-12、12-15 秒 |
| 最终模板 | 防止把分析内容原样塞给模型 | 只保留视频生成必需字段 |
| 负面提示词 | 防止生成模型常见跑偏 | 把禁区直接写入最终提示词 |

## 男装包是怎么写出来的

男装包的核心命题是：

> 不是介绍衣服，而是让观众看到“穿上这套衣服后的男人状态”。

它先把“男装带货”从传统电商表达里剥离出来：不口播、不讲价格、不放购物车、不做直播间促销，而是把视频包装成“电影角色登场片段”。

男装包的写作顺序大致是：

1. 定义单一风格：男性服装带货、15 秒、电影主角登场感、无口播、9:16 竖屏。
2. 定义核心转化心理：让男性觉得“我穿上也能帅、有气场、有身份感”。
3. 定义禁止项：不能口播、不能价格贴纸、不能汽车广告感、不能油腻霸总、不能横屏裁切、不能改衣服。
4. 定义素材入口：用户可能只给一件单品，所以必须先读图建档。
5. 定义专家角色：先是顶级男装造型师，负责把单品补成完整穿搭；再是电影摄影指导，负责镜头语言。
6. 建男装搭配库：夹克、衬衫、西装、针织、风衣、裤装等分别如何补齐鞋履、配饰、发型。
7. 建男装风格库：商务精英、黑色冷感、老钱松弛、硬朗机能、日常男友、复古文艺等。
8. 建车辆气质库：黑色轿跑、银色商务车、复古车、SUV、摩托等，用车来辅助身份和电影感，但不抢服装。
9. 建动作库：开车门、鞋落地、整理袖口、整理衣领、抬头、走向镜头、车旁定格。
10. 建摄影库：9:16 构图、低角度英雄构图、侧逆光、动机光、轮廓光、慢推、跟拍、后拉。
11. 固定 15 秒节奏：钩子 -> 细节 -> 半身气质 -> 全身穿搭 -> 主角登场 -> 车旁英雄定格。
12. 输出最终字段：标题、服装识别、锁定特征、完整穿搭、男性气质主题、场景、车辆、摄影、时间轴、负面提示词、完整视频提示词。

男装包的关键设计点是：车辆不是商品，但它是“身份氛围道具”。所以车辆必须出现，但车标、豪车炫耀和汽车广告感都被禁止。

## 女装包是怎么从男装包迁移出来的

女装包没有推翻男装包，而是做了“骨架保留 + 核心变量替换”。

保留的骨架：

- 15 秒短视频
- 9:16 竖屏
- 无口播、无字幕、无电商贴纸
- 输入单品必须扩展成完整造型
- 先读图，再造型，再场景，再摄影，最后提示词
- 10 个顺序触发词
- 风格标签库
- 气质主题库
- 摄影构图光影运镜库
- 15 秒时间轴
- 服装锁定规则
- 负面提示词
- 样片提示词

替换的核心变量：

| 男装包 | 女装包 |
|---|---|
| 男性主角登场 | 女性场景氛围感 |
| 帅气、克制、有气场 | 漂亮、自然、自信、精致、松弛 |
| 车辆身份氛围 | 服装运用场景 |
| 让男性觉得“我穿上也能帅” | 让女性觉得“我穿上也能这么漂亮” |
| 车门、鞋落地、袖口、车旁定格 | 镜前整理、鞋包首饰、妆发、回眸、坐下、走入场景 |
| 低角度英雄构图 | 显高显瘦、腰线、肩颈、裙摆动态、杂志感 |
| 男装完整穿搭 | 女装完整造型：内搭、下装、外套、鞋、包、首饰、妆容、发型、指甲、携带物 |
| 车辆气质库 | 咖啡店、通勤、约会、展览、旅行、晚餐、酒店、街角、海边等场景库 |
| 禁止油腻霸总、土味豪车 | 禁止低俗擦边、男凝视角、廉价短剧感、过度网红化 |

所以女装包的写作方法不是“重新写一个女装提示词”，而是：

> 把男装包里的“身份感道具 = 车辆”替换为“代入感场景 = 真实生活空间”，再把男性审美词库替换为女性穿搭转化词库。

## 最关键的写法：先写转化心理，再写镜头

这两个文档都不是从镜头开始写的，而是从用户购买心理开始写的。

男装：

```text
服装 -> 完整穿搭 -> 男人状态 -> 电影主角感 -> 身份气场 -> 想买
```

女装：

```text
服装 -> 完整造型 -> 女性气质 -> 生活场景 -> 变漂亮/可代入 -> 想买
```

这就是为什么文档里会反复要求：不要直接从单品图跳到视频提示词。

如果直接生成视频，模型只会拍“衣服”。  
如果先做造型、主题和场景，模型会拍“穿上这件衣服后的理想状态”。

## 10 步流程的真实作用

| 步骤 | 表面任务 | 实际作用 |
|---|---|---|
| 第一步：风格包初始化 | 确认视频类型 | 锁定边界，避免混入其他内容形式 |
| 第二步：图片读图建档 | 识别单品 | 把服装不可改变的信息写成约束 |
| 第三步：造型师激活 | 进入专家角色 | 让 AI 先补齐审美判断，不急着写镜头 |
| 第四步：单品到完整造型 | 搭配鞋包配饰等 | 把单品变成可拍摄人物 |
| 第五步：风格标签 | 归类风格 | 让场景和摄影有选择依据 |
| 第六步：气质主题 | 写人物状态 | 让视频有情绪和代入感 |
| 第七步：场景/道具匹配 | 选空间和动作 | 让衣服进入真实使用场景 |
| 第八步：摄影库 | 选构图光影运镜 | 让画面从普通展示变成短片 |
| 第九步：15 秒镜头编排 | 分时间轴 | 让信息按节奏出现 |
| 第十步：最终提示词 | 汇总输出 | 把过程分析压缩成可生成内容 |

## 最终视频提示词的固定结构

两份文档最终都会收束成这一类结构：

```text
生成一条 15 秒、9:16 竖屏、无口播、无字幕、无电商贴纸的[品类/风格]短视频。

画面风格：[电影主角登场 / 场景氛围感 / 杂志感生活方式]
人物：[成年男性/成年女性]，穿着完整[穿搭/造型]。
服装锁定：严格保持输入图片中的颜色、版型、材质、细节结构。

镜头结构：
0-2 秒：[开场钩子]
2-4 秒：[服装细节特写]
4-6 秒：[半身气质镜头]
6-9 秒：[全身造型展示]
9-12 秒：[走入场景/主角登场]
12-15 秒：[记忆点定格]

摄影风格：[9:16 构图、光影、景别、运镜]
声音：[环境音、脚步声、衣料摩擦、BGM]
禁止：[口播、字幕、电商贴纸、横屏、服装变色变款、廉价滤镜等]
```

这个结构有三个优点：

1. 生成模型能直接理解画面，不需要再读前面的分析过程。
2. 服装锁定和负面提示词被放在最终提示词里，减少跑偏。
3. 时间轴明确，15 秒视频不会变成一段散乱的氛围描述。

## 反推出来的写作原则

1. 先定“只做什么”，再定“不做什么”。边界越窄，提示词越稳定。
2. 先写用户购买心理，再写视觉风格。镜头是为转化心理服务的。
3. 输入单品不能直接进入视频生成，必须先变成完整人物和完整场景。
4. 角色设定要放在流程前段，因为它负责补齐缺失信息。
5. 词库不是装饰，而是为了让 AI 在风格、场景、动作、摄影之间做稳定匹配。
6. 负面提示词不要只放在最后校验，要直接写进最终生成提示词。
7. 样片不是随便举例，而是给模型校准输出密度、语气、镜头节奏和字段顺序。

## 两份文档最值得复用的核心

最值得复用的不是某个具体词，而是这套结构：

```text
边界
-> 规则
-> 读图
-> 专家判断
-> 完整方案
-> 风格标签
-> 场景主题
-> 摄影语言
-> 时间轴
-> 最终提示词
-> 负面提示词
-> 样例
```

以后无论拆解美妆、鞋包、食品、家居、珠宝、数字人、门店探店，都可以用这个骨架。

真正需要替换的是：

- 目标用户心理
- 产品必须保留的特征
- 适合的场景和道具
- 专家角色
- 风格标签库
- 时间轴里的动作钩子
- 负面提示词里的常见跑偏
